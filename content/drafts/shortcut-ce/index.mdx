## 들어가며
이번 포스트에서는 업무에서 필요한 수작업을 개발자도구에서 실행 가능한 짧은 코드로 대체하고, 이 코드를 버튼으로 실행할 수 있는 간단한 크롬익스텐션을 만들어본 내용을 기록하려고 합니다.

***

## 언제 수작업이 필요했는가?
현재 투입되어있는 프로젝트에서는 [Gitflow Workflow](https://www.atlassian.com/git/tutorials/comparing-workflows/gitflow-workflow)를 브랜치 전략으로 채택하여 사용하고 있습니다.  
따라서 배포를 위해 Develop 브랜치에서 Master 브랜치로 올리는 MR을 생성하고 있는데요.

이력관리 용도로 이 MR의 description 영역에 **이번 배포에는 어떤 Feature 들이 포함됩니다**를 명시하고 있으며, 해당 Feature MR의 commit hash와 jira 스토리 링크 등 사후 추적을 위해 필요한 정보를 함께 기록하고 있습니다.

현재 사용하고 있는 Gitlab이 무료라서 그런건지는 몰라도 MR을 생성할 때 자동으로 이 규칙에 맞추어 description이 작성되는 기능은 제공되지 않아 직접 description을 만들어야 하는 상황이었습니다.

***

## 첫번째. Git 명령어로 대체하기
처음엔 급한 마음에 손으로 commit history 를 복붙해서 description을 만들었지만, 감사하게도 다른 동료 분께 git 명령어를 전수(?)받을 수 있었습니다.
하지만 얼마 지나지 않아 터미널과 브라우저를 오가며 복붙을 하는 과정이 번거롭게 느껴지기 시작했습니다. 제가 git 명령어에 익숙하지 않은 탓이 컸다는 것은 비밀입니다. ㅎㅎ

## 두번째. 개발자도구로 대체하기
위에서 느낀 불편함을 해소하고자 제게 익숙한 JS로 개발자 도구 콘솔에서 description을 생성하는 짧은 코드를 만들었습니다. 브라우저에서 MR 생성과 Description 생성 & 입력이 한 번에 가능해졌기 때문에 처음에는 만족스러웠습니다.
하지만 이 방법도 역시 불편함은 있었는데요. 테스트를 위해 날려본 코드들이 개발자도구 콘솔 이력에 남아, description 생성 코드를 찾기 위해 이력을 거슬러 오르는 횟수가 많은 경우가 생겼기 때문입니다. (어이없음 주의)  

메모장에 복붙해두고 쓰는건 어떻냐 한다면... 
**Git 명령어**로 수작업을 대체했을 때 느낀 불편함과 같은 종류의 불편함을 느끼게 되지 않을까요?


## 세번째. 크롬 익스텐션에서 클릭으로 대체하기
**개발자도구**로 수작업을 대체했을 때, 위에 언급한 불편함도 물론 있었지만 크롬 익스텐션을 만들어야겠다고 결심한 데에는 다른 결정적인 이유가 있었습니다.
바로 동료 분들께 제가 쓰던 방법을 공유드릴 때 너무 날 것인 느낌이 들었기 때문인데요.
